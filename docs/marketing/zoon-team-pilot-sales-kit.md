# Zoon 商业化销售包：个人 + Team 协作

这份文档只服务一个目标：用 Zoon 当前已经具备的能力，启动商业化。当前只卖两种模式：个人模式和 Team 协作模式。不要把它讲成成熟 Team Pro、企业治理平台或 Google Docs / 飞书替代品。

## 1. 核心判断

Zoon 现在能卖的不是完整团队 SaaS，而是两种很窄的产品：

> 个人模式：用内置 Zoon Agent 或外部 Agent 完成第一次局部改稿，解决个人用户不知道怎么和 Agent 协作的问题。

> Zoon Team Pilot：帮一个文本密集团队在一个季度内跑通“多人 + 多 Agent + Markdown 改稿 + 来源可见”的真实协作流程。

第一购买理由：

> 你们已经在用 AI 改稿，但修改散在聊天窗口和飞书之间。Zoon 让 Agent 直接进入 Markdown 稿件现场，贴着原文改、评论、给建议，团队最后决定保留什么。

不要把第一卖点放在“溯源”。溯源是团队高频使用后的升级价值。第一卖点是减少复制粘贴、局部改稿和多轮合稿混乱。

## 2. 当前可承诺能力

可以承诺：

- 原生 Markdown 文档改稿。
- 多人实时协作。
- Agent 通过 HTTP 读、写、presence、评论、建议。
- 选中文本后发起快速改稿任务。
- AI / 人类内容来源可见。
- 评论、建议、保留、删除的审校流程。
- 试点结束提供人工复盘报告。

可以手动交付，但不要包装成成熟功能：

- 团队空间：用一组文档和人工 onboarding 模拟。
- 多 Agent 管理：通过明确 Agent 身份和命名约定执行。
- 溯源报告：用 marks、events 和人工整理输出。
- 外部审稿：可用分享链接，但要限定在非敏感文本场景。

不能承诺：

- 图片和富媒体排版。
- 完整 workspace / team / membership 后台。
- billing、quota、套餐限制。
- 企业级权限矩阵。
- SSO / SCIM。
- 私有部署标准流程。
- 正式审计日志和合规报告。
- SLA。

销售时要说清楚：

> 这是季度试点，不是成熟企业平台。目标是验证你们的文本改稿流程能不能从“聊天窗口复制粘贴”迁到“Agent 直接在稿件现场协作”。

## 3. ICP

当前已观察到的真实用户画像：

- 公众号作者：高频写长文，常用 AI 起草或润色，痛点是 AI 改完后还要复制回飞书 / 公众号编辑器，个人付费可能成立，但价格敏感。
- 内容编辑：每周多篇稿件，反复做局部改稿、缩短、改善表达，最容易理解“选中一段让 Agent 改”的价值；如果背后有编辑团队，可转 Team 协作。
- 产品经理写方案：写 PRD、项目方案、业务分析，文本和结构比图片更重要；如果方案需要多人审阅，是 Team 协作优先对象。

优先找：

- 长文内容团队：公众号、newsletter、SEO、访谈稿。
- 产品 / 增长团队：PRD、实验方案、复盘、用户研究。
- 咨询 / 策略团队：方案正文、汇报文稿、客户交付初稿。
- 教育 / 课程团队：讲稿、课程大纲、教案、作业点评。
- 剧本 / 口播 / 短剧团队：文本打磨频繁，图片不是核心。

合格线索标准：

| 问题 | 合格信号 |
|---|---|
| 每周文本稿件数量 | 5 份以上 |
| AI 使用频率 | 每周高频起草、改写或审校 |
| 当前工作流 | AI 输出后复制到飞书 / Docs / Notion |
| 协作人数 | 至少 2 人参与修改或审校 |
| 付费能力 | 对 ¥1,999 / 季度不立即排斥 |
| 内容形态 | 文本为主，图片不是核心工作对象 |

不优先找：

- 只把 Zoon 当个人编辑器的用户，除非能引荐团队或愿意购买个人模式。
- 小红书图文、海报、带大量图片的工作流。
- 只想一键生成低成本内容的人。
- 第一反应就要私有部署、SSO、完整审计的人。
- 不愿把真实文本流程放进试点的人。

## 4. Offer

### 个人模式

价格建议：

- ¥19.9 / 季度：轻量 quick actions。
- ¥29 / 月或 ¥99 / 季度：更多 quick actions + 少量深度改稿。

适合：个人内容编辑、产品经理、写作者、课程作者，先解决“我不知道怎么让 Agent 帮我改这一段”。

包含：

- 快速操作：修复语法、改善表达、缩短。
- 局部追问：对选中文本提问。
- 局部改稿：让 Agent 只改这一段，不重写全文。
- 保留 / 删除 AI 修改。
- 基础 AI / 人类来源可见。

目标：

> 让个人用户在第一次打开文档后 10 分钟内完成 first agent collaboration。

不要把个人模式卖成 token 包。用户买的是“开箱即用的 Zoon Agent”和更顺的改稿动作，不是模型额度。

### Team 协作模式

价格：¥1,999 / 季度

适合：已经理解痛点、愿意自己上手的小团队。

包含：

- 1 个手动配置的团队改稿空间。
- 10-30 份真实文本协作文档。
- 2-5 名人类协作者。
- 1-3 个 Agent 身份。
- 3 套改稿 prompt 或任务模板。
- 45 分钟 onboarding。
- 1 次试点结束复盘。

目标：

> 让团队完成至少 10 份真实 AI 协作文档，并判断 Zoon 是否减少复制粘贴、局部追问和合稿混乱。

不要在这个阶段卖：

- Team Pilot Plus。
- Team Pro。
- Team Scale。
- 企业治理年费。
- 私有部署。
- 未定义清楚的“团队平台”。

## 5. Demo 脚本

Demo 不要从“这是一个 Markdown 编辑器”开始。直接从用户当前痛点开始。

### 5 分钟 Demo

1. 打开一篇 AI 味重的 Markdown 草稿。
2. 选中一段表达不自然的文字。
3. 点击快速操作：改善表达 / 缩短 / 修复语法。
4. 让 Agent 在原文旁边给出修改或建议。
5. 展示保留 / 删除，强调“人最后决定哪句留下”。
6. 展示 Agent presence 或评论，强调“Agent 在同一份文档现场工作”。
7. 展示来源标记，强调“至少知道哪段是 AI 写入、哪段是人保留”。

Demo 结束句：

> 你现在不用把这一段复制给 AI，再把结果粘回飞书。Agent 直接贴着这段原文改，你只决定保留什么。

### 15 分钟 Demo

在 5 分钟 Demo 基础上增加：

- 第二个人加入协作。
- 第二个 Agent 只做评论，不直改。
- 展示一处建议被保留、一处建议被删除。
- 展示试点复盘会记录哪些数据：文档数、Agent 写入数、保留/删除比例、典型评论。

不要展示：

- 图片工作流。
- 未完成的团队后台。
- 企业权限。
- 复杂路线图。

## 6. 销售访谈脚本

开场：

```text
我不是想问你喜不喜欢一个新工具。我想判断你们团队有没有一个值得付费解决的 AI 改稿流程问题。

我们看到很多内容和产品团队已经高频用 AI 改稿，但流程还是：AI 输出，复制到飞书，人工改，局部拿不准，再复制给 AI，再粘回来。Zoon 想验证的是：Agent 如果能直接进入稿件现场改、评论、给建议，这个流程是否值得团队付费。
```

核心问题：

1. 你们每周有多少份文本稿件 / PRD / 方案会用 AI 改？
2. 这些文档最后交付给谁？
3. 现在 AI 改稿结果怎么进入飞书 / Google Docs / Notion？
4. 哪一步最烦：复制粘贴、局部追问、多人审稿、版本混乱，还是最终合稿？
5. 有几个人参与最终修改或审校？
6. 文档里图片和排版重要吗，还是文本本身最重要？
7. 如果 Zoon 帮你们跑一个季度的 Markdown 改稿协作流程，Agent 直接贴原文改，团队保留/删除修改，试点 ¥1,999，你愿意买吗？
8. 如果不愿意，阻碍是价格、工作流、缺图片、团队习惯，还是信任？
9. 如果愿意，第一批 10 份真实文档是什么？
10. 谁能批准这笔试点费用？

不要问：

- “你觉得 Zoon 怎么样？”
- “你愿不愿意免费试试？”
- “如果未来有完整团队版你会不会用？”

这些问题不会产生付款信号。

## 7. 反对意见处理

### “我们已经有飞书 / Google Docs”

```text
Zoon 不是替代飞书。飞书是最终协作文档和交付空间。Zoon 先解决中间改稿流程：Agent 不再只停在聊天窗口里，而是能直接贴着 Markdown 原文改、评论、给建议。试点成功后，你们仍然可以把最终稿迁回飞书。
```

### “我们需要图片”

```text
这轮试点不适合图片重的内容。Zoon 当前最适合文本密集工作流，比如长文、PRD、方案正文、讲稿、剧本。图片和排版不是这轮承诺范围。
```

### “为什么不免费试？”

```text
可以自己免费创建文档体验。但季度 Team Pilot 不是卖账号，是帮你们跑一个真实团队改稿流程，包括 onboarding、模板、真实文档、复盘。我们要验证的是它能不能进入你们的真实工作，而不是多试一个工具。
```

### “¥1,999 贵了”

```text
如果只是个人 AI 写作工具，确实不该这么定价。这个试点卖的是团队流程：减少复制粘贴、减少局部追问、减少合稿混乱。如果你们一周有 5-10 份 AI 参与稿件，一个季度跑通一个流程，¥1,999 是合理的验证成本。
```

### “担心安全”

```text
这轮试点建议先选非敏感或脱敏文本。Zoon 当前不承诺企业级权限、SSO、私有部署和合规审计。我们先验证协作流程价值，企业治理能力会放到后续 Team Scale design partner 阶段。
```

### “我们想要完整 Team Pro”

```text
现在不卖成熟 Team Pro。我们先用季度 Team Pilot 跑真实流程。只有当 3-5 个试点都证明团队愿意复用和续费，才把重复需求产品化成 Team Pro。
```

## 9. 季度交付节奏（首月跑通）

### Week 0：成交与准备

- 确定团队负责人。
- 确定 10 份真实文档类型。
- 确定参与人和 Agent 身份。
- 明确哪些内容不能放入 Zoon。
- 准备 3 套 prompt：改表达、缩短、指出问题。
- 建立试点文档清单。

### Week 1：Onboarding

- 45 分钟 walkthrough。
- 跑通第一份真实文档。
- 完成 first agent collaboration。
- 记录用户第一次卡住的位置。
- 记录第一句“这有用”或“这不行”的原话。

### Week 2：真实改稿

- 完成至少 3-5 份协作文档。
- 观察复制粘贴是否减少。
- 记录最常用动作：直改、评论、建议、快速操作。
- 记录哪里仍然会回到飞书或聊天窗口。

### Week 3：团队协作

- 邀请第二个人类协作者。
- 邀请第二个 Agent 或切换 Agent 角色。
- 测试一个评论型 Agent 和一个直改型 Agent。
- 观察团队是否真的使用来源标记和审校动作。

### Week 4：复盘与续费

- 统计协作文档数。
- 统计 Agent 写入 / 评论 / 建议次数。
- 统计保留 / 删除比例。
- 记录最有价值场景。
- 记录最大阻塞。
- 提出续费或 Team Pro design partner 方案。

## 11. 第一周行动清单

第 1 天：

- 从现有 5 个用户里挑出最接近团队场景的 2 个。
- 约 30 分钟观察，不帮忙，看他们完成一篇真实稿件。
- 记录 first agent collaboration 卡点。

第 2 天：

- 整理 20 个文本密集团队线索。
- 每个线索写清楚：团队类型、文档类型、AI 使用频率、可能负责人。

第 3-4 天：

- 发出 20 条私信或邮件。
- 目标不是 demo，而是约 30 分钟访谈。

第 5 天：

- 完成 3 场访谈。
- 至少对 1 个团队明确报价 ¥1,999 / 季度。

第 6-7 天：

- 复盘反对意见。
- 调整 demo。
- 继续报价，不接受“先免费试试”作为成功信号。
